## ⚠️ Day-1 must-do items (before any partner sees this)

These are the things that are unsafe to leave as-is in production.

### 1. Rotate every shared secret currently in Railway

The current production secrets were generated by me (Claude) during the deploy. **Treat them as compromised by default.** Rotate, redeploy.

```bash
# Generate fresh values
./scripts/generate-prod-secrets.sh > new-secrets.env
# Paste each into Railway → API service → Variables, then save
# Trigger a redeploy. Verify smoke test passes.
# Delete new-secrets.env from disk.
```

Secrets to rotate:

- `JWT_ACCESS_SECRET`, `JWT_REFRESH_SECRET`
- `PII_ENCRYPTION_KEY` ⚠️ rotating this requires re-encrypting every PII column in `applications` + `credit_enrichments` (decrypt-with-old, encrypt-with-new migration script). **Do this BEFORE any real applicant submits a form.**
- `PII_HASH_SECRET` — rotating breaks every analytical join. Do once at handover, never again.
- `*_WEBHOOK_SECRET` × 4 (Pixie, MiCamp, EazePay App, HighSale) — mirror new values into the corresponding upstream system.

### 2. Rotate every demo account password + every ingestion PAT

Demo accounts seeded with `Demo!1234`. The 7 portfolio PATs (`epi_pk_*`) printed during the deploy were emailed/shared with Brodie. **Rotate all** before opening the URL to partners.

```bash
# Per-user: log in as admin, change password from /admin
# Per-org PAT: revoke via SQL, then re-run the seed
psql "$DATABASE_URL" -c "UPDATE api_tokens SET revoked_at = now() WHERE revoked_at IS NULL;"
pnpm --filter api db:seed:portfolio-orgs   # prints fresh PATs once
```

### 3. Move Postgres + Redis to private DNS

The deploy uses Railway's **public TCP proxy** for both. This was a temporary workaround because the API container couldn't resolve `redis.railway.internal` / `postgres.railway.internal` on the first boot.

Fix: enable IPv6 egress on the API service in Railway (Settings → Networking) and switch `DATABASE_URL` + `REDIS_URL` back to:

```
DATABASE_URL=${{Postgres.DATABASE_URL}}
REDIS_URL=redis://default:${{Redis.REDIS_PASSWORD}}@${{Redis.RAILWAY_PRIVATE_DOMAIN}}:6379
```

Once that's live, **delete the public TCP proxies** on both services to close the public-facing port.

### 4. The 3 outstanding security fixes from the pre-handover audit

Each is small. Land in your first PR.

- **`SEC-004`** — webhook signature uses `JSON.stringify(req.body)` instead of the raw body. Register `@fastify/raw-body` and switch to `req.rawBody`. Affects `webhook-signature.middleware.ts`, `eazepay-app.routes.ts`, `highsale.routes.ts`.
- **`SEC-009`** — `/customers` list returns credit score / income / propensity to **VIEWER + INVESTOR** scope. Add a scope filter in `customer.routes.ts` line 53–149.
- **`SEC-010`** — refresh-token rotation upgrades scope from `investor` to `standard`. Persist scope on the `RefreshToken` row, reuse on rotation (`auth.service.ts:76`).

Full audit: [`docs/reviews/`](docs/reviews/) (the 2026-05-14 pre-handover audit will land here in a follow-up commit).
